# 財務儀表板 WealthPulse - 規格文件

## 1. 專案概述

- **名稱**：WealthPulse（財富脈動）
- **類型**：響應式網頁應用程式（個人財務儀表板）
- **目標用戶**：個人使用者，追蹤整體財務狀況
- **核心功能**：整合收入、支出、負債，展示 Net Worth 趨勢與成長百分比

## 2. 技術堆疊

| 層面 | 技術 |
|------|------|
| 前端 | React + Vite + Tailwind CSS |
| 後端 | Node.js + Express |
| 資料來源 | Notion API（支出）+ Yahoo Finance API（股票）+ 手動輸入（薪水、房租、房貸）|
| 部署 | Vercel (前端) + Railway (後端) |

## 3. 資料來源

### 3.1 收入（手動輸入或 API 串接）
| 來源 | 類型 | 更新頻率 |
|------|------|----------|
| 薪水 | 月薪 | 每月手動更新 |
| 股票配息 | 股息 | 依配息公告更新 |
| 房租 | 租金收入 | 每月手動更新 |

### 3.2 股票
- 使用 Yahoo Finance API 抓取個股股價
- 用戶輸入股票代碼（如 2330.TW、AAPL）
- 計算：持有股數 × 現價 = 股票市值
- 股票配息可從 API 或手動輸入

### 3.3 支出（Notion）
- 串接現有 Notion 支出資料庫
- 自動計算每月支出總額

### 3.4 負債
| 類型 | 說明 |
|------|------|
| 房貸餘額 | 手動輸入或 API（若有） |
| 信用卡負債 | 手動輸入 |

## 4. 計算公式

```
Net Worth = 總資產 - 總負債

總資產 = 薪水餘額 + 股票市值 + 股票配息 + 房租收入
總負債 = 房貸餘額 + 信用卡負債

月變化率 = (本月 Net Worth - 上月 Net Worth) / 上月 Net Worth × 100%

成長狀態：
- 正數：顯示綠色 +X.X%
- 負數：顯示紅色 -X.X%
- 零：顯示灰色 0%
```

## 5. API 端點

### 5.1 財務數據
- `GET /api/finance/summary` - 取得財務摘要（Net Worth、月變化）
- `GET /api/finance/trend` - 取得趨勢數據（歷史記錄）
- `GET /api/stock/prices` - 抓取股票現價

### 5.2 使用者資料
- `POST /api/user/income` - 新增收入來源
- `POST /api/user/liability` - 新增負債
- `GET /api/user/assets` - 取得資產配置

### 5.3 Notion 串接
- `GET /api/notion/expenses` - 從 Notion 取得支出資料

## 6. 前端頁面

### 6.1 儀表板首頁（Dashboard）
- **總覽卡片**：Net Worth + 成長百分比（大字顯示）
- **資產分布圖**：圓餅圖（薪水、股票、房租）
- **負債分布圖**：圓餅圖（房貸、信用卡）
- **月變化趨勢圖**：折線圖（過去 6-12 個月）
- **快速摘要**：
  - 本月收入：$XXX
  - 本月支出：$XXX
  - 本月結餘：$XXX

### 6.2 資產頁面（Assets）
- 薪水帳戶餘額
- 股票清單（代碼、名稱、股數、現價、市值、配息）
- 房租收入記錄

### 6.3 負債頁面（Liabilities）
- 房貸餘額 + 月付額
- 信用卡負債

### 6.4 設定頁面（Settings）
- Notion API Key 串接
- 股票代碼管理（新增/刪除）
- 收入/負債手動輸入

## 7. UI/UX 設計

### 7.1 色彩方案
- 主色：#10B981（Emerald，成長、正向）
- 負向色：#EF4444（Red，下降）
- 背景：#0F172A（深色模式）/ #F8FAFC（淺色模式）
- 卡片：#1E293B / #FFFFFF
- 文字：#F1F5F9 / #1E293B

### 7.2 趨勢指示
```
+5.2%  📈 綠色（成長）
-3.1%  📉 紅色（下降）
 0.0%  ➡️ 灰色（持平）
```

### 7.3 響應式
- Mobile：第一銀行網銀風格，卡片式陳列
- Desktop：儀表板網格式布局

## 8. 資料儲存

### 選項 A：Notion（推薦）
- 使用者已在 Notion 記帳，可擴充為財務資料庫
- 優點：統一介面、易於查看

### 選項 B：後端資料庫（SQLite/PostgreSQL）
- 獨立儲存，隱私性更高
- 需部署資料庫服務

## 9. 安全考量

- 需登入（Google OAuth）才能查看
- Notion API Token 存於環境變數
- Yahoo Finance 為公開 API，無需認證
